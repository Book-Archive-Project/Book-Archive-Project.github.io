Project Goals

Our goal is to create a dynamic, responsive website for the book "Journal of a Pandemic Year". The site will display media from various sources in an intuitive way such that visitors can interact with media reflecting topics discussed in the book, as well as other events that occurred during 2020.

This link will take you to the project backlog where all of the major goals and accomplishments are documented.

Approach

Our approach for this project was to create a wordpress site, to develop a theme and plugin to interface with Zotero, and to create the responsive and dynamic functionality.

The typical user experience is as follows:

Once on the website, the End User will have the ability to browse by date. There will be a page for each date that has media sources. On each date page, various sources will be displayed including web sources, images, video, audio, and more. The user can then interact with these sources however they choose to, and navigate between pages.

Tools

Various tools and languages used include Php, PhpStorm, Github, Html, Css, Javascript, phpDocumentor, MAMP/XAMPP, and Postman.

Video

Our strategy will be to utilize Wordpress to create a website for the book. We will create plugins in php that interact with the Zotero and Group 11 databases to achieve the required dynamic functionality as well as allowing for the display of desired media.

The typical user experience is as follows:

Once on the website, the End User will have the ability to search for a specific date, or view the entire site at once. If a date is searched for, a page with all of the corresponding sources will be shown to the user. The user then can click on one of the sources to interact with it more ie. play media, open link to external source, etc.. The user can then choose to go to the next day, search for another day, or view the entire site. When viewing the entire site, all of the dates with available sources are listed in order, grouped by month, and allow the user to select one from the list to view.
